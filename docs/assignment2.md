# The Second Week's Assignment
第2週の課題「課題2」について説明します。

___
- グループごとにROSパッケージ（複数のROSノードが連携した1つの機能）を作成してもらいます。

- まず、グループごとに課題提出用のブランチを作成してください。グループ名は話し合って決めてください。グループの誰か1人が作成し、他の人はそのブランチに切り替えてください。
```
《記法》
$ git checkout -b feature/グループ番号_グループ名
《実例》
$ git checkout -b feature/0_baymax
```

- 次に、ディレクトリー「laboratory_experiments_2026」の中にROSパッケージ「assignment_package」を作成してください。グループの誰か1人が作成・`push`し、他の人は`pull`しましょう。

- そして、「伝言ゲーム」のようなトピック通信を実現してください。
    - それぞれのメンバーが1つずつROSノードを作成し、それらを連携させます。<span style="color: #CC0066;">特定のメンバーのみがプログラミングを担当することは避けてください。</span>また、自分のノードは自分で`push`してください。
    - それぞれのノードにはsubscriberとpublisherの両方の機能を実装することになります。
    - 全てのノードを1つのlaunchファイルで1度に起動できるようにしてください。
    - 最初のトピックはコマンド`$ ros2 topic pub`で出版してください。また、最後のトピックはコマンド`$ ros2 topic echo`で購読してください。

- 実現するトピック通信についてはグループで話し合い、決めてください。「伝言ゲーム」のように少しずつ文字列を変化させても良いですし、「尻取り」のように言葉を繋げても良いですし、…。乱数を用いて確率的に変化させても面白そうです。ロボットおよび人工知能の研究者としては言葉の創発なども興味深いですね。

- 実装後は「そのようなトピック通信を実現しようと考えた理由」や「トピック通信を実現するために工夫した部分」などをレポートとして文書ファイルにまとめてください。
    - 執筆を担当した人、そして各人が担当した部分を明記してください。
    - 論文や図書などの文献を参考した場合は、その部分を明記してください。また、AIを利用した場合も、その部分を明記してください。
    - レポートの用紙サイズはA4縦とします。
    - 図表を使用しても構いません。
    - PDF形式で提出してください。
    - ファイル名は「グループ番号_グループ名.pdf（例：0_baymax.pdf）」でお願いします。

- 最後に、ROSパッケージ「assignment_package」の中にreadme.mdを作成してください。readmeファイルに書いてある手順でROSパッケージを実行し、評価します。念の為、正常に実行できた場合の出力例を示してください。markdownファイルの書き方は[ココ](https://docs.github.com/ja/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax)などを参考にしてください。

- 評価可能な状態になったら、グループの代表者がMicrosoft Teamsのグループチャットで鈴木拓央＜SUZUKI Takuo＞に連絡してください。代表者以外のメンバーも提出状況を確認できるよう、他のメンバーもグループに加えてください。レポート（PDFファイル）もグループチャットで提出してください。

- 締切は次回の実験実施日の12時00分までとします。

___
以上です。よろしくお願いします。

［Hint］ROSノードやROSトピックの状態を確認する必要があるので、ROSツール（rqt関連）やROSコマンド（ros2 topic関連）などを調査し、利用してみましょう。
